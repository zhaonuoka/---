> 去 draw.io 画完后截图放这里

### 架构说明

本系统采用前后端分离架构：

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端层 | React + TypeScript + Ant Design | 游客端和管理后台，运行在浏览器中 |
| 后端层 | Python FastAPI | 提供 RESTful API 和 WebSocket 接口 |
| AI层 | 通义千问 API + ChromaDB | 大模型推理 + RAG 知识库检索 |
| 数据层 | SQLite | 存储对话日志、知识库文档、数字人配置 |
| 语音层 | faster-whisper + Edge-TTS | 语音识别与合成，后端内置 |

### 完整对话流程

```
用户说话 → 前端录音 → STT语音识别 → RAG检索知识库 → 通义千问生成回答
→ TTS语音合成 → 前端播放语音 + 数字人口型同步 → 自动记录日志 + 情感分析
```

## 2. 技术选型

### 2.1 为什么选 FastAPI

| 对比项 | FastAPI | Flask | Django |
|--------|---------|-------|--------|
| 性能 | ✅ 异步支持，高并发 | ❌ 同步，性能一般 | ❌ 重量级 |
| API文档 | ✅ 自动生成 Swagger UI | ❌ 需手动配置 | ❌ 需手动配置 |
| WebSocket | ✅ 原生支持 | ❌ 需扩展 | ❌ 需扩展 |
| 学习成本 | ⭐⭐ 中等 | ⭐ 低 | ⭐⭐⭐ 高 |

**结论：** FastAPI 性能好、自带API文档、支持WebSocket，适合AI类应用。

### 2.2 为什么选 ChromaDB + BAAI/bge-small-zh-v1.5

- **ChromaDB**：轻量级向量数据库，无需单独部署，适合比赛场景
- **BAAI/bge-small-zh-v1.5**：中文向量模型，比默认的 all-MiniLM-L6-v2 中文检索效果好很多

### 2.3 为什么选通义千问 API

- 国产大模型，免费额度高（新用户7000万Token + 高校认证3000万）
- 中文理解能力强，适合景区问答场景
- API 兼容 OpenAI 格式，调用方便

### 2.4 语音技术选型

| 模块 | 技术 | 理由 |
|------|------|------|
| STT（语音识别） | faster-whisper tiny | 开源免费，中文识别强，比原版Whisper快4倍 |
| TTS（语音合成） | Edge-TTS | 免费不限量，中文发音自然，"晓晓"音色好 |

## 3. 模块设计

### 3.1 RAG知识库模块

**功能：** 用户提问 → 检索相关景点资料 → 拼入提示词 → 调用通义千问 → 返回回答

**检索流程：**
```
① 用户提问 → ② 两次检索（景点库 + 指南库）
③ 合并去重 → ④ 拼入提示词模板
⑤ 调用通义千问 API → ⑥ 返回回答
```

**为什么做两次检索：**
- 第一次搜景点数据（22个景点的位置/参数/文化内涵）
- 第二次单独搜指南和问答数据（历史/门票/路线）
- 合并结果，指南数据放前面，确保不会漏掉

**知识库规模：** 104 条知识片段
- 景点数据：88 条（22个景点 × 4个维度）
- 指南知识：4 条（历史/文化/门票/游览）
- 推荐路线：4 条
- 常见问答：8 条

### 3.2 语音识别模块

**技术：** faster-whisper tiny 模型
**流程：** 接收音频文件 → 保存临时文件 → whisper识别 → 返回文字
**特点：** 本地运行，无需联网，单次识别<1秒

### 3.3 语音合成模块

**技术：** Edge-TTS（微软免费引擎）
**流程：** 接收文字 → 调用Edge-TTS → 返回base64编码的MP3音频
**语音：** zh-CN-XiaoxiaoNeural（晓晓，自然女声）

### 3.4 推荐路线模块

**逻辑：** 接收用户兴趣标签（历史/自然/亲子/禅修/摄影）→ 匹配路线标签 → 返回最佳路线

**4条推荐路线：**
| 路线 | 时长 | 适合标签 |
|------|------|---------|
| 历史文化深度游 | 6小时 | 历史、文化、摄影 |
| 自然风光休闲游 | 5小时 | 自然、风景、禅修 |
| 亲子欢乐游 | 4小时 | 亲子、趣味 |
| 禅意深度体验游 | 7小时 | 禅修、文化、静心 |

### 3.5 情感分析模块

**流程：** 每次对话结束后 → 用通义千问分析用户问题情感 → 标记正面/中性/负面 → 存入数据库

**用途：** 管理后台的"游客感受度报告"和"数据大屏"的数据来源。

## 7. 部署说明

### 环境要求
- Python 3.10+
- Node.js 18+
- 4GB RAM（推荐8GB）
- Windows / Linux / macOS

### 启动步骤
```bash
# 1. 安装后端依赖
cd backend
pip install -r requirements.txt

# 2. 初始化数据库和知识库
python init_kb.py

# 3. 启动后端
python main.py

# 4. 新终端，启动前端
cd frontend
npm install
npm run dev
```

### 注意事项
- 首次启动会自动下载中文向量模型（约100MB）
- 需提前申请通义千问 API Key
- 演示时确保电脑已联网（需调用通义千问API）
